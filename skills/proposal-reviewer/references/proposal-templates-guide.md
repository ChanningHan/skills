## 模板选择建议

根据方案类型快速匹配：

- **前端方案** → 前端方案模板
- **后端方案** → 后端方案模板
- **架构方案** → 架构方案模板
- **产品方案** → 产品方案模板
- **重构方案** → 重构方案模板
- **性能优化方案** → 性能优化模板

## 模板组合使用

对于复杂方案，可以组合使用多个模板：

1. **主模板**：选择最符合方案类型的模板
2. **补充模板**：参考其他模板的评审要点，补充评审维度
3. **自定义调整**：根据实际情况调整评审维度

### 组合示例

- **前端重构方案**：重构方案模板（主）+ 前端方案模板（补充）+ 性能优化模板（补充）
- **全栈产品方案**：产品方案模板（主）+ 前端方案模板（补充）+ 后端方案模板（补充）
- **架构升级方案**：架构方案模板（主）+ 性能优化模板（补充）

## 生成自定义模板

如果现有模板无法满足评审需求，可以根据方案特点生成定制化模板：

1. **参考现有模板**：选择一个最接近的模板作为基础
2. **分析方案特点**：识别方案的特殊评审需求
3. **定义评审维度**：根据方案特点定义或调整评审维度
4. **生成模板内容**：基于现有模板结构，生成符合需求的模板内容

### 生成原则

- **保持结构一致性**：遵循标准评审报告结构（概述、角色评审、汇总、行动项、结论）
- **明确评审维度**：每个角色应有清晰的评审维度和评分标准
- **保持格式规范**：使用统一的 Markdown 格式和评分体系
